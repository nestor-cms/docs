VCE Class API

VCE is a utility class that contains properties and methods used by all components throughout the Nestor system, and which can be easily accessed with a custom component. During the initiation phase, all of the core engine class objects are added to the global $vce object, which makes it much simpler to access them in another component.

All of the $vce properties are added at run-time, during the page build process.

Create Datalist

Creates a datalist in the database to store data for your component(s). Datalists can have associated items that can be added using this method. You don’t need to provide values for all of the attributes below, as the function will fill in ‘null’ values to those you don’t provide. However, your datalist won’t have much value if there is no data associated with it.

You can create a datalist hierarchy by associating a datalist with a item, which basically turns the item into its own datalist. The $attributes array for this scenario will differ from a non-hierarchical datalist.

Refer to the Datalist section for more information about datalist architecture.

create_datalist($attributes)

Parameters

$attributes Array Information used to create the datalist
Potential members of the $attributes array
parent_id Int if you are creating a nested datalist, this is the ID of the parent datalist
item_id Int (Optional) Allows you to make an item into its own datalist, references an item_id in the datalist_items table
component_id Int (Optional) the component to which the data will be associated
user_id Int (Optional) user_id of the user to whom the datalist will be associated
datalist String name of the datalist, can be used to search for the datalist so make sure it’s unique
sequence Int (Optional) Order in which the datalist will be rendered in a list
aspects Array (Optional) key/value pairs to assign to the datalist_meta table, represent datalist metadata
hierarchy Array (Optional) Indexed array of values. Allows you to specify levels of hierarchy, such as states and cities. Note that the items array for a hierarchical scenario will also need to be hierarchical (see below)
items Array (Optional) key/value pairs that represent items that will be associated with the datalist
Potential members of the aspects array
[key] String Key for the key/value pair(s) you are adding to the datalists_meta table. You can include multiple key/value pairs.
Potential members of the hierarchy array
[index] String Values in an indexed array. Allows you to specify levels of hierarchy, such as states and cities. The first value will relate to the top parent datalist, then the next level will be applied as the name of the children datalist(s) in the datalists_meta table.
Potential members of the items array
[index] Array Holds multiple arrays, each of which represents a new datalist item
Members of each item array
[key] String Key for the key/value pair(s) associated with the item. This data will be entered into the datalist_items_meta table. You can include multiple key/value pairs.
Members of each item array - hierarchical
[key] String Key for the key/value pair(s) associated with the item. This data will be entered into the datalist_items_meta table. You can include multiple key/value pairs.
items Array A new items array that represents an item to associate with the parent item/datalist. Members of this array would be key/value pairs, and a potential new items array if there was a level below this item level.

Return

Int datalist ID

Example

//non-hierarchical

$attributes = array (
  'parent_id' => '0',
  'item_id' => '0',
  'component_id' => '1',
  'user_id' => '0',
  'sequence' => '1',
  'datalist' => 'State_datalist',
  'aspects' => array (
    'name' => 'State',
    'type' => 'select'
  ),
  'items' => array (
    array (
      'name' => 'CA',
      'state_flower' =>
      'golden poppy'
    ),
     array (
      'name' => 'WA',
      'state_flower' =>'rhododendron'
    )
  )
);
$vce->create_datalist($attributes);
//hierarchical

$attributes = array (
  'parent_id' => '0',
  'item_id' => '0',
  'component_id' => '0',
  'user_id' => '0',
  'sequence' => '1',
  'datalist' => 'State_datalist',
  'aspects' => array (
    'name' => 'State',
    'type' => 'select'
  ),
  'hierarchy' => array ('State', 'City'),
  'items' => array (
    array (
      'name' => 'WA',
      'state_flower' => 'rhododendron'
    ),
    array (
      'name' => 'CA',
      'state_flower' => 'golden poppy',
      'items' => array (
        array (
          'name' => 'Sacramento',
          'mayor' => 'Darrell Steinberg'
        ),
        array (
          'name' => 'Los Angeles',
          'mayor' => 'Eric Garcetti'
        )
      )
    )
  )
);

$vce->create_datalist($attributes);

Get Datalist

Find the datalist(s) associated with the specified attribute, and return its metadata. Function will look for passed values in the following order, and then will stop parsing $attributes array at the first non-null value (or value pairs) it finds. Will then search datalist table for the datalist(s) with matching value(s). So it makes sense to only pass one of the following values or combos, based on what you are using to search.

  1. component_id
  2. datalist_id
  3. user_id AND datalist
  4. datalist
  5. user_id
  6. item_id

This function can be filtered to remove certain datalists from the results set.

get_datalist($attributes)

Parameters

$attributes Array Contains attributes to use to retrieve datalists
Potential members of the $attributes array
component_id Int Component ID to search for in datalists table
datalist_id Int Datalist ID to search for in datalists table
user_id Int User ID to search for in datalists table
datalist String Datalist name to search for in datalists_meta table
item_id Int Item ID to search for in datalists table. Note: this is not the item_ID in the datalists_items table

Return

Boolean Return false if no parameters specified
Array Array containing all matching datalists, including associated metadata. Returned array will be keyed using datalist_id of each datalist
Members of the returned array
datalist_id String ID of datalist
parent_id String ID of parent datalist
item_id String ID of corresponding item in datalist_items
component_id String ID of associated component
user_id String ID of user data is associated with
sequence String Order in which datalist appears in a list
datalist String Unique name for the datalist
[key] String Key / value pairs of associated metadata, common keys are ‘name’ and ‘type’
hierarchy Array Indexed arrays with values representing hierarchy

Example

$datalists = $vce->get_datalist( array('component_id' => '11') );
if (!empty($datalists)) {
  foreach ($datalists as $datalist_info) {
    $datalist_id = $datalist_info['datalist_id'];
    $datalist = $datalist_info['datalist'];
    $parent_id = $datalist_info['parent_id'];
    $item_id = $datalist_info['item_id'];
    $user_id = $datalist_info['user_id'];
    $component_id = $datalist_info['component_id'];
    $datalist_name = $datalist_info['datalist'];
  } }

Get Datalist Items

Return all items from the datalist_items table that match the specified attribute(s), as well as extra information about the datalist. The values in the returned dataset vary based what attributes(s) you pass in for the search.

The method will look for passed values in the following order, and will stop parsing $attributes array at the first non-null value(s) it finds. Will then search datalist or datalist_meta table for the datalist(s) with matching value(s). So it makes sense to only pass one of the following values or combos, based on what you are using to search.

  1. datalist_id
  2. name
  3. parent_id AND item_id
  4. user_id AND datalist
  5. datalist
  6. user_id
  7. component_id

The function will return data from the first matched datalist it finds. If there are multiple datalists that match your search parameter, you’ll get the items associated with the first datalist found by the search query. This would normally be the one with the lowest datalist_id.

get_datalist_items($attributes)

Parameters

$attributes Array Contains attributes to use to retrieve datalists
Potential members of the $attributes array
datalist_id Int Datalist ID to search for in datalists table
name String Name to search for in datalists_meta table
parent_id Int Parent ID to search for in datalists table
item_id Int Item ID to search for in datalists table. Note: this is not the item ID in the datalists_items table
user_id Int User ID to search for in datalists table
datalist String Datalist name to search for in datalists_meta table
component_id Int Component ID to search for in datalists table

Return

Boolean Returns false if no datalists returned from query
Array Returns array of datalist items and their associated metadata

Values in the returned Array vary based what attributes(s) you pass in for the search.

Members of the returned array using one of the following search parameters: datalist_id, component_id, datalist, user_id, parent_id & item_id or user_id & datalist
datalist_id String Datalist ID
parent_id String ID of datalist Parent
item_id String ID of associated Item
component_id String ID of associated Component
user_id String ID of associated User
sequence String Order in which this datalist will display in a rendered list of datalists
items Array Holds arrays, each with information about a datalist item
Members of items array
[datalists_item_id] Array Info about an individual datalist item
Members of each [datalists_item_id] array
item_id String ID of datalist item
sequence String Order in which this datalist item will display in a rendered list of items
[key] String This array will contain all key/value pairs from the datalists_items_meta table associated with the datalist item.

Members of the returned array when searching on name
id String ID of datalists_items record
datalist_id 
String
 ID of associated datalist
meta_key String meta_key from datalists_meta table with value of [name] you are searching for
meta_value String meta_value field associated with meta_key = [name] you are searching for
minutia String minutia from datalists_meta table
items Array Holds arrays, each with information about a datalist item
Members of items array
[datalists_item_id] Array Info about an individual datalist item
Members of each [datalists_item_id] array
item_id String ID of datalist item
sequence String Order in which this datalist item will display in a rendered list of items
[key] String This array will contain all key/value pairs from the datalists_items_meta table associated with the datalist item.

Members of the returned array when searching on item_id
items Array Holds array with information about the datalist item. This record set is much smaller than the other two, as it will only ever return information about one datalist item.
Member of items array
[datalists_item_id] Array Info about an individual datalist item
Members of each [datalists_item_id] array
item_id String ID of datalist item
sequence String Order in which this datalist item will display in a rendered list of items
[key] String This array will contain all key/value pairs from the datalists_items_meta table associated with the datalist item.

Example

$attributes = array('datalist_id' => '10');
$options = $vce->get_datalist_items($attributes);
$insert = "options: ";
if (isset($options['items'])) {
  foreach ($options['items'] as $each_option) {
    $insert .= '<option value="' . $each_option['item_id'] . '"';
    $insert .= '>' . $each_option['name'] . '</option>';
  }
}
echo '<select>'. $insert . '</select>';

Manage Cron Task

This is a very flexible method that allows you to add, update or delete a cron task in the site_meta table. During the page load process, the Site class loads all of the cron tasks checks for those with expired timestamps. If the task is expired, Site will call the indicated method and then delete the cron task from the system. If the method returns a response formatted as an $attributes array that manage_cron_task can accept, Site will call manage_cron_task with the returned array.

Note that when testing cron task management, if you use date() to set the timestamp, the task will fire the very next time you load a page and your task may get garbage collected by the system. You may want to set your timestamps far in the future when testing cron task management, at least until you are ready to test the method you’ve assigned to the task.

manage_cron_task($attributes)

Parameters

$attributes Array The details of the cron task to be added, updated or deleted
Values in the $attributes array:
timestamp Int Timestamp to indicated when task expires
action String Action to take on the task, values can be ‘add’, ‘update’, or ‘delete’
id String ID of cron_task record in the site_meta table, optional if sending ‘add’ as the action
value String JSON encoded string containing an array, holds other properties for the cron task
Values in the (encoded) value array:
component String Name of the component that contains the function called by this task
method String Name of the method called by this task
properties Array key/value pairs to associate with the task
Values in the (encoded) properties array:
[key] String Key to the supplied value. Can add as many key/value pairs as needed to this array. An example would be the component_id for the component you are calling.

Return

Int Returns cron_task_id from site_meta table if successful add
Boolean Returns true if successful update or delete
No returned value if not successful

Example

//add cron task
$cron_task['action'] = 'add';
$cron_task['timestamp'] = time();
$cron_task['value'] = json_encode(
  array (
    'component' => 'StateManager',
    'method' => 'update_state_flowers',
    'properties' => array (
      'component_id' => '10'
    )
  )
);

$cron_task_id = $vce->manage_cron_task($cron_task);
//update cron task

$cron_task['action'] = 'update';
$cron_task['timestamp'] = time();
$cron_task['id'] = $cron_task_id;
$cron_task['value'] = json_encode(
  array (
    'component' => 'StateManager ',
    'method' => 'update_state_flowers',
    'properties' => array (
      'component_id' => '145',
    )
  )
);

$vce->manage_cron_task($cron_task);

update_datalist_item($attributes)

Parameters

$attributes Array Attributes that you want to update for the specified datalist item
Potential members of the $attributes array
item_id Int Item ID in the datalists_items table. Note there is also an item_id in the datalists table, which this is not
relational_data Array data that you want to update in the datalists table
meta_data Array key/value pairs to update in the datalists_meta table.
Potential members of the relational_data array
datalist_id Int datalist ID in the datalists_items table
sequence Int The order in which this datalist item appears in any rendered list
Potential members of the meta_data array
[key] String Key for the key/value pair(s) you are updating in the datalists_items_meta table

Return

Boolean Return true if successful update, false if not successful

Example

$attributes = array (
  'item_id' => '21',
  'relational_data' => array(
  'datalist_id' => '9',
  'sequence' => '5'),
  'meta_data' => array (
  'name' => 'KS',
  'state_flower' => 'prairie rose')
);

$updated = $vce->update_datalist_item($attributes);
