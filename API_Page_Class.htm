Page Class

The Page Class contains properties and methods used to build out the final page. It first determines what page is being requested, by reading the URL and pulling out either a component ID or Recipe url value. Page then fills a multi-dimensional component array based on parent-child relationships in the database – these entries and relationships are created by the Recipe and by end users adding content to Pages. Last, Page reads the Recipe and compares it to the component array, and determines what content should be displayed based on user role and other settings.

Methods

Name/Syntax

Description

Private build_content

Builds content from components. Called from __construct(), is given a list of components and the recipe, and then orders the components and

recursively calls on them to add their content to the $content object.

This can’t be called by another component directly. However it does interact with components in the page being built and contains hooks where a component can insert functionality.

Use the display_components() method to call build_content() from another component.

Private get_components ($page_id, $requested_id, $components)

Finds all components upstream of the component $requested, and then fills in all sub_components. Returns multi-dimensional array of all components related to a specific page.

This can’t be called by another component directly. However it does interact with components in the page being built and contains hooks where a component can insert functionality.

The get_parents() method allows you to partially emulate this method from another component.

Private get_sub_components ($current_id, $parent_id, $components, $sub_components = array(), $sub_url = false, $full_object = false)

Finds all components downstream of the $current_id and adds to a multi-dimensional array.

This can’t be called by another component directly. However it does interact with components in the page being built and contains hooks where a component can insert functionality.

Use the get_children() method to call get_sub_components() from another component.

Display Components

Calls build_content to allow you to see what the final page output will be. Note: if you want to have recipe_components / add_component work for components sent to this method, you need to send along a requested_id with the recipe. $requested_id should be the component_id from where you want the sub_recipe to allow add_content. The recipe should include the parent component of the requested_id.

display_components ($components, $recipe = null, $requested_id = null, $linked = false, $recipe_tracker = array())

Parameters

$components

Array

Array of the components that you want to display. This would be along the lines of $page->components

$recipe

Array

Recipe for the components

$requested_id

Int

The requested ID that goes with the $recipe. Generally is the component_id of the first component in the recipe

$linked

Boolean

A value used in the build_content method to track recursion. False by default

$recipe_tracker

Array

An array that acts as a placeholder when traversing the $components array. Set to blank by default

Return

No return value

Example

$vce->page->display_components ($vce->page->components, $vce->page->recipe,

$vce->page->components[0]->component_id);
